# UX Documentation

Denne mappe beskriver Cortex' brugeroplevelse.

Formålet er at sikre, at alle kliniske workflows opleves ens, uanset om konsultationen handler om knæsmerter, hoste, diabetes eller depression.

UX-dokumentationen beskriver **hvordan lægen arbejder** – ikke hvordan systemet implementeres.

## Struktur

- **Wireframes** beskriver skærmopbygning.
- **UX Principles** beskriver de generelle designprincipper.
- Kliniske pathway-specifikationer placeres under `docs/clinical/pathways/`.

## Designhierarki

Vision foundation
↓
Engineering and clinical governance
↓
Product requirement or proposal
↓
Architecture and design guidance
↓
Prototype
↓
Kode

Den universelle konsultationsworkflow er defineret i `docs/vision/WF-001-The-Consultation-Workflow.md`.
