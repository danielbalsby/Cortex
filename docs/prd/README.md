# Product Requirements Documents

PRD'er beskriver den konkrete brugeroplevelse og funktionalitet for Cortex.

Hvor RFC'er beskriver arkitektur og principper, beskriver PRD'er, hvordan et klinisk workflow skal fungere fra lægens perspektiv.

Et PRD skal kunne læses af:

- Praktiserende læger
- UX-designere
- Frontend-udviklere
- AI-udviklere
- Produktansvarlige

Et PRD må ikke beskrive implementering.

Det beskriver produktets adfærd og brugeroplevelse.

Alle referenceworkflows starter som et PRD.
