# Cortex Review Process

Every significant milestone is reviewed before release.

Reviews are independent from implementation.

The purpose is not to find bugs.

The purpose is to evaluate long-term quality.

---

## Review types

### Architecture Review

Evaluates:

- maintainability
- abstraction
- technical debt
- scalability

---

### Product Review

Evaluates:

- workflow
- UX
- cognitive load
- feature scope

---

### Clinical Design Review

Evaluates:

- consultation flow
- clinician experience
- documentation quality

---

### Clinical Validation

Evaluates:

- evidence
- guideline alignment
- regional applicability

This review requires qualified clinicians.

---

## Decision states

Every review ends with one of:

- Approved
- Approved with observations
- Changes required

---

Reviews become permanent project documentation.
